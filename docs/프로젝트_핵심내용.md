## 프로젝트 개요

**Schedule Master**는 개인 및 팀의 일정, 업무, 약속을 통합 관리하는 웹 애플리케이션입니다.

- **프론트엔드**: React + Vite (SPA)
- **백엔드**: Express.js (API 서버)
- **데이터베이스**: Supabase (PostgreSQL)
- **AI**: Google Gemini 2.0 Flash
- **배포**: Vercel — https://schedulemaster-one.vercel.app

## 데이터 구조 개요

### sm_tasks (업무)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | uuid | PK |
| `title` | text | 업무 제목 |
| `task_date` | date | 시작일 |
| `due_date` | date | 마감일 (기간 업무) |
| `task_time` | time | 업무 시각 |
| `duration` | integer | 소요 시간 (분) |
| `priority` | text | `low` / `medium` / `high` |
| `category` | text | 업무 카테고리 |
| `status` | text | `todo` / `in_progress` / `done` / `review` |
| `project_id` | uuid | 프로젝트 FK (null = 독립 업무) |

### sm_appointments (약속)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | uuid | PK |
| `title` | text | 약속 제목 |
| `date` | date | 약속 날짜 |
| `start_time` | time | 시작 시각 |
| `end_time` | time | 종료 시각 |
| `color` | text | 캘린더 표시 색상 |

### sm_users (사용자)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | uuid | PK (Supabase Auth 연동) |
| `email` | text | 이메일 |
| `display_name` | text | 표시 이름 |
| `avatar_url` | text | 아바타 이미지 URL |

### sm_chip_presets (칩 프리셋)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | uuid | PK |
| `label` | text | 칩 표시 텍스트 |
| `type` | text | `title` / `category` 등 |
| `value` | text | 실제 적용 값 |

## 문서 탐색 방법

1. 이 파일(`프로젝트_핵심내용.md`)을 먼저 읽어 전체 구조를 파악합니다.
