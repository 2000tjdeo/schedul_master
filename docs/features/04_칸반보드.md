# 04. 칸반보드

업무를 상태(status)별 컬럼으로 시각화하고 드래그앤드롭으로 관리하는 기능입니다.

---

## 컴포넌트 목록

| 컴포넌트 | 경로 | 용도 |
|----------|------|------|
| `KanbanBoard` | `src/components/KanbanBoard.jsx` | 데스크탑용 칸반보드 |
| `MobileKanbanBoard` | `src/components/MobileKanbanBoard.jsx` | 모바일용 칸반보드 |
| `KanbanCard` | `src/components/KanbanCard.jsx` | 개별 업무 카드 |

---

## 컬럼 구성

| 컬럼 ID | 표시 이름 | 설명 |
|---------|----------|------|
| `todo` | 할 일 | 아직 시작하지 않은 업무 |
| `in_progress` | 진행 중 | 현재 작업 중인 업무 |
| `done` | 완료 | 완료된 업무 |
| `review` | 검토 중 | 검토 또는 승인 대기 중 |

```js
// 컬럼 정의
const COLUMNS = [
  { id: 'todo',        label: '할 일' },
  { id: 'in_progress', label: '진행 중' },
  { id: 'done',        label: '완료' },
  { id: 'review',      label: '검토 중' },
];
```

## KanbanBoard.jsx — 데스크탑

### 레이아웃

```
KanbanBoard
├── 컬럼: 할 일 (todo)
│   └── KanbanCard × N
├── 컬럼: 진행 중 (in_progress)
│   └── KanbanCard × N
├── 컬럼: 완료 (done)
│   └── KanbanCard × N
└── 컬럼: 검토 중 (review)
    └── KanbanCard × N
```

### 드래그앤드롭 구현

HTML5 Drag and Drop API를 사용합니다.

```js
// 드래그 시작
const handleDragStart = (e, taskId) => {
  e.dataTransfer.setData('taskId', taskId);
};

// 드롭 시 상태 변경
const handleDrop = (e, newStatus) => {
  const taskId = e.dataTransfer.getData('taskId');
  updateTask(taskId, { status: newStatus });
};
```

## KanbanCard.jsx — 업무 카드

### 카드 구성 요소

```
KanbanCard
├── 우선순위 색상 인디케이터 (좌측 세로 바)
├── 업무 제목
├── 카테고리 태그
├── 날짜 정보 (task_date / due_date)
├── 담당자 아바타 (sm_users.avatar_url)
└── 드래그 핸들 아이콘
```

### 우선순위 색상

| 우선순위 | 색상 | Tailwind 클래스 |
|---------|------|----------------|
| `low` | 초록 | `bg-green-400` |
| `medium` | 노랑 | `bg-yellow-400` |
| `high` | 빨강 | `bg-red-500` |

```jsx
const priorityColor = {
  low:    'bg-green-400',
  medium: 'bg-yellow-400',
  high:   'bg-red-500',
};

<div className={`w-1 h-full rounded-l ${priorityColor[task.priority]}`} />
```

### 담당자 아바타

```jsx
{task.assignee && (
  <img
    src={task.assignee.avatar_url}
    alt={task.assignee.display_name}
    className="w-6 h-6 rounded-full"
    title={task.assignee.display_name}
  />
)}
```

## 상태 변경 — store/taskStore.js

칸반보드의 드래그앤드롭 완료 시 `updateTask`를 호출하여 Supabase와 전역 상태를 동시에 업데이트합니다.

```js
// store/taskStore.js
const updateTask = async (id, changes) => {
  // 1. Zustand 전역 상태 즉시 업데이트 (낙관적 업데이트)
  set(state => ({
    tasks: state.tasks.map(t => t.id === id ? { ...t, ...changes } : t)
  }));

  // 2. Supabase DB 동기화
  const { error } = await supabase
    .from('sm_tasks')
    .update(changes)
    .eq('id', id);

  if (error) {
    // 실패 시 롤백
    console.error('업무 업데이트 실패:', error);
  }
};
```

---

## 데이터 흐름

```
드래그앤드롭 완료
        │
        ▼
handleDrop(taskId, newStatus)
        │
        ▼
store/taskStore.js → updateTask(id, { status: newStatus })
        │
        ├─ Zustand 상태 즉시 반영 (낙관적 업데이트)
        └─ Supabase sm_tasks.update({ status }) 동기화
```
