# 02. 업무·약속 생성

업무(Task)와 약속(Appointment)을 생성하는 통합 모달과 관련 컴포넌트를 다룹니다.

---

## 컴포넌트 구조

```
UnifiedCreateModal
├── 탭 선택 (업무 / 약속)
├── NLStrip (자연어 입력 + 음성 버튼)
├── Quick Menu (칩 팔레트)
│   ├── 제목칩 (title)
│   ├── 날짜칩 (date)
│   ├── 시간칩 (time)
│   └── 소요시간칩 (duration)
└── 폼 필드 (수동 입력 영역)
```

| 컴포넌트 | 경로 | 역할 |
|----------|------|------|
| `UnifiedCreateModal` | `src/components/UnifiedCreateModal.jsx` | 통합 생성 모달 진입점 |
| `NLStrip` | `src/components/NLStrip.jsx` | 자연어 입력창 + 음성 인식 버튼 |

---

## NLStrip — 자연어 입력

사용자가 자유롭게 텍스트를 입력하면 Gemini AI가 파싱하여 폼 필드를 자동으로 채웁니다.

```jsx
// 자연어 입력 예시
"내일 오후 3시에 팀 미팅 1시간"
// → title: "팀 미팅", date: 내일, time: "15:00", duration: 60
```

- 입력창 오른쪽에 마이크 버튼 배치 (`useSpeech` 훅 연결)
- 입력 완료(Enter 또는 버튼 클릭) 시 `parseNLWithGemini` 호출

---

## Quick Menu — 칩 팔레트

`sel` state로 어떤 칩이 명시적으로 선택되었는지 추적합니다.
칩을 클릭하면 해당 필드가 활성화되고, 다시 클릭하면 선택 해제됩니다.

### 칩 종류

| 칩 | `sel` 값 | 동작 |
|----|---------|------|
| 제목칩 | `'title'` | `sm_chip_presets`에서 불러온 프리셋 제목 선택 |
| 날짜칩 | `'date'` | 인라인 달력 팝업으로 날짜 선택 |
| 시간칩 | `'time'` | 시간 드롭다운 선택 |
| 소요시간칩 | `'duration'` | 15·30·60·90·120분 선택 |

```js
// sel state 관리 예시
const [sel, setSel] = useState(null);

const handleChipClick = (chipType) => {
  setSel(prev => prev === chipType ? null : chipType);
};
```

## handleNLParsed — 자연어 파싱 결과 처리

Gemini가 파싱한 결과를 받아 폼 상태를 업데이트하고,
**시간 또는 소요시간이 변경될 때 `end_time`을 자동 계산**합니다.

```js
const handleNLParsed = (parsed) => {
  setForm(prev => {
    const next = { ...prev, ...parsed };

    // time 또는 duration이 바뀌면 end_time 자동 계산
    if (next.task_time && next.duration) {
      const [h, m] = next.task_time.split(':').map(Number);
      const totalMin = h * 60 + m + Number(next.duration);
      const endH = String(Math.floor(totalMin / 60) % 24).padStart(2, '0');
      const endM = String(totalMin % 60).padStart(2, '0');
      next.end_time = `${endH}:${endM}`;
    }

    return next;
  });
};
```

## handleSubmit — Supabase 제출 처리

제출 직전에 업무/약속 타입에 따라 **불필요한 필드를 제거**합니다.

```js
const handleSubmit = async () => {
  if (type === 'task') {
    const { allDay, end_time, ...taskPayload } = form;
    // sm_tasks 테이블에 삽입
    await supabase.from('sm_tasks').insert(taskPayload);
  } else {
    const { allDay, duration, priority, category, ...apptPayload } = form;
    // sm_appointments 테이블에 삽입
    await supabase.from('sm_appointments').insert(apptPayload);
  }
};
```

> **주의**: `allDay`와 `end_time`은 UI 전용 필드로, Supabase 스키마에 없으므로 반드시 제거 후 제출해야 합니다.

## Supabase 필드 매핑

### sm_tasks (업무)

| 폼 필드 | DB 컬럼 | 타입 | 설명 |
|---------|---------|------|------|
| `title` | `title` | text | 업무 제목 |
| `startDate` | `task_date` | date | 시작일 |
| `endDate` | `due_date` | date | 마감일 (기간 업무) |
| `time` | `task_time` | time | 시작 시각 |
| `duration` | `duration` | integer | 소요 시간 (분) |
| `priority` | `priority` | text | `low` / `medium` / `high` |
| `category` | `category` | text | 업무 카테고리 |
| ~~allDay~~ | — | — | UI 전용, 제출 전 제거 |
| ~~end_time~~ | — | — | UI 전용, 제출 전 제거 |

### sm_appointments (약속)

| 폼 필드 | DB 컬럼 | 타입 | 설명 |
|---------|---------|------|------|
| `title` | `title` | text | 약속 제목 |
| `date` | `date` | date | 약속 날짜 |
| `start_time` | `start_time` | time | 시작 시각 |
| `end_time` | `end_time` | time | 종료 시각 (자동 계산) |
| `color` | `color` | text | 캘린더 표시 색상 |

## 전체 데이터 흐름

```
사용자 입력 (텍스트 / 음성 / 칩 선택)
        │
        ▼
NLStrip → parseNLWithGemini(text)
        │
        ▼
handleNLParsed → form state 업데이트 + end_time 자동 계산
        │
        ▼
Quick Menu 칩으로 수동 보완 (sel state)
        │
        ▼
handleSubmit → 불필요 필드 제거
        │
        ├─ sm_tasks.insert(taskPayload)
        └─ sm_appointments.insert(apptPayload)
```
