# 07. 프로젝트 개념 설계 (미구현 — 로드맵)

> **참고:** 현재 미구현. 설계 방향 문서.

---

## 핵심 원칙

> **project_id = null이면 독립 스케줄** — 프로젝트 없는 업무/약속은 그대로 사용 가능.

---

## 추가될 데이터 구조

### sm_projects (신규 테이블)

| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | int | PK |
| title | text | 프로젝트명 |
| description | text | 설명 |
| color | text | 프로젝트 색상 |
| start_date | date | 시작일 |
| end_date | date | 종료일 |
| created_by | int | 생성자 (sm_users.id) |

### sm_tasks 추가 컬럼

| 컬럼 | 타입 | 설명 |
|---|---|---|
| project_id | int | sm_projects.id (nullable — 독립 업무면 null) |
| parent_task_id | int | 부모 업무 id (서브태스크 구조, nullable) |
| order | int | 같은 레벨 내 정렬 순서 |
| points | int | 스토리포인트 (Flow의 숫자) |

---

## 뷰별 동작

| 뷰 | 프로젝트 업무 표시 방식 |
|---|---|
| 캘린더 | 기간 bar (이미 TaskBar로 구현됨) |
| 칸반 | 프로젝트 탭 전환으로 필터링 |
| 리스트 | 계층형 트리 (펼침/접기) |
| 간트 *(신규)* | 주차별 타임라인 |

---

## 구현 우선순위

1. **1단계** — `sm_projects` 테이블 생성 + 프로젝트 선택 UI (업무 생성 모달)
2. **2단계** — 리스트 뷰에서 계층형 트리 렌더링 (parent_task_id 활용)
3. **3단계** — 간트 차트 뷰 추가 (주차 기준 타임라인)
